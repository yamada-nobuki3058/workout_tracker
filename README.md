# トレーニング記録・分析アプリ

## ■ アプリ概要

日々のトレーニング記録を「面倒で続かない」と感じているユーザー向けに、
筋トレ・有酸素・身体データを一元管理し、
**「入力の手間を最小化しながら、成長を可視化・分析できる」トレーニング管理アプリです。**

単なる記録ではなく、**「記録 → 可視化 → 改善」までを一貫してサポートすること**を目的としています。

## ■ 主な機能

### ● トレーニング記録機能

* 筋トレ（重量・回数・複数セット）
* 有酸素（距離・時間・複数セット）
* 日付ごとのWorkout管理

📸 **スクリーンショット（トレーニング入力画面）をここに掲載予定**

---

### ● プリセット機能（強み）

* トレーニングメニューを事前登録可能
* 種目だけでなく**セット単位（重量・回数など）までテンプレート化**
* 記録時に即反映され、入力は「調整のみ」で完結

📸 **スクリーンショット（プリセット反映後の画面）をここに掲載予定**

---

### ● 初期値の自動入力

* 前回の記録を優先して自動入力
* なければプリセットの値を使用
* ユーザーは微調整のみで記録可能

---

* 種目ごとの最大重量・総重量の推移

* 身体データ（体重・体脂肪）の推移
* ベスト / ワーストの表示

📸 **スクリーンショット（グラフ分析画面）をここに掲載予定**

---

### ● ダッシュボード

* 直近の記録表示
* ミニグラフで状態を可視化
* AIによる簡易アドバイス表示

📸 **スクリーンショット（ダッシュボード画面）をここに掲載予定**

---

### ● カレンダー機能

* トレーニング実施日の可視化

---

### ● 種目管理

* 共通種目 + ユーザー独自種目に対応

## ■ DB設計の特徴

* 種目はExerciseMasterで一元管理（表記ゆれ防止）
* 筋トレ / 有酸素を分離し、NULLを排除
* プリセットは中間テーブルで柔軟に構成
* **プリセット専用のSetテーブルにより、セット単位の再現が可能**
* 片側種目はフラグで管理し、集計時に補正

---

## ■ 工夫した点（強み）

* プリセット + 前回値による「提案型入力」により、入力時間を大幅削減
* トレーニングの継続を意識したUX設計
* データの可視化だけでなく、改善につながる分析機能を実装
* Docker / CI/CDなど実務を意識した構成

## ■ 制作背景

トレーニング記録は入力の手間が大きく、継続が難しいと感じたため、
「できるだけ簡単に記録でき、かつ成長を実感できるアプリ」を目指して開発しました。
